Earnstrack

Earnstrack is a full-stack transaction tracker for e-wallet and printing-service operations. The project combines a React and TypeScript frontend with an ASP.NET Core backend, JWT authentication, role-based access control, service fee management, CSV export, and PostgreSQL persistence.

Problem

What the app is solving

Small service-based businesses often manage e-wallet, printing, and seller activity across separate notes or spreadsheets. That makes it harder to track totals, standardize service charges, and review work by period or operator.

Scattered records

Manual tracking creates inconsistency across transaction types and reporting periods.

Fee management

Service charges need to be configurable without rewriting the app for every pricing change.

Role-based usage

Admins and sellers need different levels of visibility and control inside the same product.

Solution

How Earnstrack responds

The app centralizes transaction entry, dashboard reporting, service fee configuration, authentication, and export workflows in one responsive interface.

1. Sign in and protect routes

Users authenticate with email and password, then the frontend stores the session and guards routes for authenticated access.

2. Record transactions by service

Separate forms support E-Wallet and Printing workflows while still feeding a single reporting model.

3. Review dashboard totals

The dashboard summarizes daily, weekly, and monthly totals along with recent transactions and status breakdowns.

4. Admin configuration

Settings provide appearance control, service fee management, user administration, and transaction export.

Stack

PostgreSQL or Supabase Postgres
Migrations for schema changes
Audit and transaction entities

Highlights

Portfolio talking points

These are the strongest points to emphasize when presenting the project to recruiters, clients, or collaborators.

Operational focus

The project is not just a landing page or admin shell. It models concrete business workflows for e-wallet and printing services.

Full-stack delivery

The app spans UI, API, database schema, authentication, and test coverage instead of stopping at frontend visuals.

Role-aware design

Admin-only actions such as user management and settings help show that the product accounts for different user responsibilities.

Room to grow

The existing structure leaves space for features like inventory, richer reporting, and more service categories.
